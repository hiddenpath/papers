\subsection{Action Layer Decomposition}

A critical insight is that the action layer comprises two subcomponents:

\paragraph{Orchestration component.}
The orchestration component is responsible for capability routing, provider selection, policy evaluation, and environment selection.

\paragraph{Execution component.}
The execution component is responsible for API invocation, tool execution, error handling, and result normalization.

This decomposition maintains clear separation of concerns. The orchestration layer makes decisions, such as which provider or environment to use, while the execution layer performs actions, such as invoking APIs and returning normalized results.

When mapped to the logical model in \cref{eq:logical-model}, the orchestration component corresponds to $P$, and the execution runtime corresponds to $E$.

\subsection{Defining Statement}

\begin{quote}
\emph{The execution layer is the action interface between AI cognition and the real-world capability ecosystem.}
\end{quote}

This definition captures the essential role of the layer: mediating between what agents decide to do and how those decisions are realized through capability execution.
